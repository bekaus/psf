Logging

Usage

psf provides logging to stderr via the PSF_LOG macro: 

 PSF_LOG(logINFO) << "Hello" << username << "no endl, will be appended automatically";

You can set a plateu logging level: 

 #undef PSF_LOG_MAX_LEVEL
 #define PSF_LOG_MAX_LEVEL psf::logWARNING

This can be used to set a global logging level or to construct a more fine grained structure (for example, different logging levels per file or code segment).

The logging macro PSF_LOG ensures, that logging code will only be compiled into the final binary, if it corresponds to the set plateu logging level.

As a consequence, don't use the Log API directly, but only via the provided macros.

Provided logging levels

These are the available logging levels ordered from the highest to the deepest: 

 logERROR, logWARNING, logINFO, logDEBUG, logDEBUG1, logDEBUG2, logDEBUG3, logDEBUG4

Turning off logging

If logging and the accompanying code is not desired in the final binary, a special logging level is provided: 

 logNO_LOGGING

Set it as the plateu logging level to disable logging.

Don't use logNO_LOGGING as a paramter to the logging macro.
